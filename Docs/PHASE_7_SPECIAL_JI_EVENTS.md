# 阶段 7：冲锋鸡、责任鸡与乌骨鸡

## 本阶段交付

- 新增乌骨鸡识别：八筒由统一规则计算器识别，不在牌桌逻辑中重复写死。
- 新增鸡统计范围枚举：`HandOnly`、`HandAndMeld`、`HandAndDiscard`、`HandMeldAndDiscard`。
- 新增可配置单位值：幺鸡、翻鸡、乌骨鸡、冲锋鸡、乌骨冲锋鸡、责任鸡和乌骨责任鸡。
- 新增冲锋鸡事件：全局首个符合条件的幺鸡或乌骨鸡弃牌由服务器记录。
- 新增责任鸡事件：首个目标鸡被其他玩家碰或明杠时，记录获得者和责任支付者。
- 新增特殊鸡定向分：责任玩家向获得者直接支付配置分值，并保持四家严格零和。
- 公共牌桌和结算结果同步特殊鸡事件；结算界面显示普通鸡、特殊鸡和杠分的独立变化。
- 所有新增配置写入规则规范字符串和哈希，开局后不可被客户端篡改。

## 默认事件语义

1. 一条始终可以作为冲锋鸡和责任鸡目标；其单位值可设置为 0 以关闭计值。
2. 启用乌骨鸡且对应开关开启时，八筒也可成为首个目标鸡。
3. 当前版本按一局全局首个目标鸡判定，最多生成一个冲锋鸡事件和一个对应责任鸡事件。
4. 冲锋鸡单位加入弃牌者的鸡数；责任鸡使用定向分，不通过鸡数差额间接模拟。
5. 被碰或杠的弃牌进入副露后不会在弃牌统计范围中重复计数。

## 规则配置字段

- `JiCountingScope`
- `BasicJiValue`
- `FlippedJiValue`
- `WuGuJiValue`
- `ChongFengJiValue`
- `WuGuChongFengJiValue`
- `ZeRenJiValue`
- `WuGuZeRenJiValue`
- `bWuGuCanChongFeng`
- `bWuGuCanZeRen`

所有数值由规则快照规范化至 `0..16`，再乘以 `JiScore` 形成实际责任分。

## 明确保留的扩展点

- 每名玩家分别判定首个冲锋鸡。
- 一局允许多个冲锋鸡或责任鸡。
- 冲锋鸡被碰或杠后的归属策略。
- 金鸡倍率、未听牌反赔、黑八必须杠和特殊杠倍率。
- 黄庄查叫后对鸡分和责任分的二次调整。
