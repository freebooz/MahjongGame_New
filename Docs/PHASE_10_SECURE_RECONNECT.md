# 阶段 10：安全断线重连与私有快照恢复

## 本阶段交付

- 活跃房间断线时不再释放座位，房间座位只切换为离线状态。
- `UGuiyangRoomManager` 记录断线时间，并按锁定规则中的 `ReconnectTimeoutSeconds` 校验重连窗口。
- 客户端使用登录子系统中仅存在于进程内存的会话凭据发起认证。
- 服务器按玩家 ID 首次绑定凭据摘要，后续认证使用常量时间摘要比较。
- 凭据原文不会写入 PlayerState、GameState、房间状态、重连快照、SaveGame 或日志。
- 相同玩家 ID 已在线时拒绝第二个控制器接管座位。
- 重连成功后恢复房间状态、公共牌桌、本人私有手牌、当前可操作列表和结算弹窗。
- 私有快照新增 `LastAcceptedClientSequence`，用于恢复客户端动作序号基线。

## 断线流程

```text
连接断开
  -> GameMode::Logout
  -> 活跃房间调用 MarkDisconnected
  -> 座位、累计分和手牌继续保留
  -> bOnline = false
  -> 牌桌由阶段 9 的服务器托管继续推进
```

开局前普通大厅房间仍按主动离开流程清理，避免永久占用未开始的房间。

## 重连流程

```text
新 PlayerController BeginPlay
  -> 从 LoginSubsystem 读取内存会话
  -> Server_AuthenticateSession
  -> 服务端验证格式、摘要和重复在线状态
  -> RoomManager::ReconnectPlayer
  -> 恢复原座位
  -> Client_RestoreReconnectSnapshot
```

重连快照包含：

- 房间规则、累计分、局数、庄家和在线状态。
- 当前公开牌桌阶段、弃牌、副露、倒计时和剩余牌数。
- 仅该玩家可见的手牌、副露及最后接受的客户端序号。
- 当前该玩家可执行的胡、碰、杠或过候选。

## 安全边界

- 当前 MVP 使用首次绑定模式：玩家 ID 第一次成功认证时绑定该进程内会话凭据。
- 该模式支持同一客户端进程内的网络中断、切网和重新连接。
- 应用彻底退出后的跨进程安全恢复需要正式账号服务或平台令牌，当前版本不会把会话令牌持久化到本地。
- 超过重连保留时间后服务器拒绝恢复；座位仍可由服务器托管至牌局结束，避免牌局状态被强制破坏。

## 验证结果

- UE 5.8 Editor Development 构建成功。
- `GuiyangMahjong` 自动化测试 25 项全部成功。
- 多局生命周期测试新增断线座位保留、房间映射保留、在线状态恢复和剩余时间断言。
- 安全测试确认重连快照不存在 Token、Secret 或 Password 字段。
- 超时测试确认私有快照携带最后接受的动作序号。
