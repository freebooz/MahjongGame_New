# 阶段 11：最终大结算与稳定排行榜

## 本阶段交付

- 新增 `FMahjongFinalPlayerResult`：名次、座位、昵称和累计总分。
- 新增 `FMahjongFinalSettlementResult`：房间号、完成局数和最终玩家列表。
- 房间管理器提供纯数据构建函数，所有服务器和重连路径使用同一排序逻辑。
- 达到配置总局数后，GameMode 向所有玩家可靠广播一次最终大结算。
- 重连到已结束房间的玩家会重新收到相同最终排行榜。
- 移动端复用单局结算弹窗展示最终排名，并隐藏“下一局”按钮。

## 排名规则

```text
第一排序键：累计总分，从高到低
第二排序键：座位编号，从低到高
名次：排序后依次为 1、2、3、4
```

同分时使用座位编号保证排序稳定，使四个客户端、服务器日志和重连恢复得到完全一致的结果。

## 广播约束

- 只有房间生命周期为 `Settlement` 且 `CurrentRound >= RoundCount` 时才生成最终结果。
- GameMode 使用房间状态序号防止同一最终结算重复广播。
- 最终排行榜不包含手牌、弃牌细节、登录凭据或重连凭据。
- 单局结算仍先展示详细基础分、鸡分、责任分和杠分；随后切换为最终累计排行榜。

## 移动端显示

- 标题：最终大结算。
- 摘要：已完成局数和房间号。
- 玩家行：名次、座位、昵称和带正负号的累计分。
- 最终结算隐藏“下一局”，保留返回大厅入口。
- 普通单局结算重新显示“下一局”，避免复用弹窗后状态残留。
