# 阶段 9：操作倒计时与服务器自动托管

## 本阶段交付

- 规则配置新增：`bEnableTimeoutAutoPlay`、`TurnTimeoutSeconds`、`ReactionTimeoutSeconds`。
- 配置规范化：出牌超时限制为 3 至 120 秒，响应超时限制为 3 至 60 秒。
- 规则快照：所有超时配置进入规范字符串和哈希，开局后不可修改。
- 公共牌桌状态：同步本次操作总时长和服务器截止时间。
- 玩家回合超时：服务器优先打出刚摸到的牌，不可用时选择排序后的末张牌。
- 响应窗口超时：仅为尚未提交动作的候选座位自动补“过”。
- 防陈旧回调：超时处理必须同时匹配 `RoundId`、`TurnId` 和 `Phase`。
- GameMode 定时器：阶段或回合变化时重新装配；同一响应窗口中的部分响应不会延长倒计时。
- 移动端 HUD：使用 `GameState.GetServerWorldTimeSeconds()` 实时显示剩余秒数。

## 安全约束

- 客户端不能请求摸牌，也不能选择服务器托管时要打出的牌。
- 自动操作走牌桌权威入口并推进服务端动作序号。
- 上一局、上一回合或上一阶段的定时器即使迟到，也只返回失败，不修改牌桌。
- 已提交的胡、碰、杠或过不会被超时处理覆盖。
- 响应超时后仍使用正常优先级裁决；自动过牌不改变已声明动作的优先级。

## 状态流程

```text
PlayerTurn 超时
  -> 服务端选择牌并出牌
  -> WaitingForAction 或下一玩家 PlayerTurn

WaitingForAction 超时
  -> 未响应座位自动 Pass
  -> 正常优先级裁决
  -> Settlement、吃碰杠后回合或下一玩家回合
```

  - 响应窗口自动过牌。
  - 弃牌不被错误声明。
  - 同一超时回调不能重复执行。
